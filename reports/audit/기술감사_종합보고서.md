## 한 줄 결론

**현재 Chronos2는 “환경 forcing을 이해하는 운영형 causal 예보 시스템”이 아니라, 강한 자기상관 기반 probabilistic autoregressive model에 가깝다.**

- 평균 NSE·coverage는 일부 타깃에서 양호하지만, **극단·급변·worst 지점**에서는 성능이 크게 붕괴한다.
- **고위험 자동 경보 시스템으로의 운영 배포는 아직 부적합**하다. 보조 대시보드·참고 예보 수준은 가능하다.

## 3. 13개 감사 항목별 요약

### [1] Covariate Ignorance — **확인됨**

- 운영 코드 `forecast.py`에서 **`cov=None` 고정** → 강우·유량·수위·댐 방류 미사용.
- 공변량 ablation: 평균 ΔNSE **-0.006**, 개선 **1/4** 셀.
- Counterfactual(공변량 변경)은 운영 모델 기준 **예측 변화 0** (no-op).

**판단**: endogenous dominance + **covariate ignorance** 동시 존재. causal forcing 학습 증거 없음.

---

### [2] Dynamic Uncertainty — **부분적 / fake static width 의심**

- day5 width–error 상관: 0.41~0.61 (일부 동적성).
- 급변 top 5%에서 width ratio 1.9~4.3배 확대 → **일부 event-sensitive**.
- 그러나 급변 시 **pred_delta vs obs_delta 상관이 전 타깃 음수** → spike onset 적응 부족.
- conformal 후 crossing 증가 → **usable uncertainty로 보기 어려움**.

---

### [3] Extreme Regime — **붕괴 확인**

- 상위 5% TOC/DO/T-P: NSE **음수**.
- large-error top 10%: cov80_raw **0.26~0.34** (과신).
- **interpolation에는 강하고 extreme extrapolation에는 약함.**

---

### [4] Leakage — **다수 위험 신호**

| 우선순위 | 항목 | 위험 |
|----------|------|------|
| **P0** | `light_fill()` 양방향 보간 | test context에 미래값 유입 가능 |
| **P0** | `final_eval` origin 240h vs context 512h | val/train context 혼입 |
| P1 | AIHub Test 미사용 (내부 70/10/20) | holdout 정의 모호 |
| P1 | `replay._resolve_threshold` test obs 혼입 | alert metric optimistic |
| P2 | Legacy imputed NSE | 원논문 대비 낙관적 채점 |

**Gold standard 경로**: `src/system/replay.py` (origin `i_va+512`, 인과 conformal).

---

### [5] Persistence Dependency — **강함**

- forecast vs persistence 상관: 극단 subset에서도 **0.86~0.99**.
- median |q50 − persistence|: 대부분 타깃에서 작음.
- 갑작스러운 변화 시 예측이 실제 변화를 **따라가지 못함** (delta 상관 ≤0).

---

### [6] Calibration vs Sharpness — **inflation 위험**

- cov80: raw ~0.75 → cal ~0.81 (목표 정렬).
- 그러나 Winkler·width 동시 개선 여부 불명확, **구간 crossing** 발생.
- **coverage 향상이 interval inflation + integrity 손상**일 가능성.

---

### [7] Spatial Robustness — **편차 큼**

- TOC: 21개 지점이 persistence 미만.
- T-P worst-5%: NSE **-0.430**.
- 수계 편향: 한강 T-P median NSE **0.19**, 금강·낙동은 상대 양호.

---

### [8] Quantile Integrity — **raw 양호 / cal 불량**

- raw crossing: **0%**.
- cal80 crossing: 타깃별 **1.7~2.9%**, horizon1 **최대 8.1%**.

---

### [9] Legacy Reproduction — **충실도 제한**

| 구분 | legacy 원본 | deep_see 재구현 |
|------|-------------|-----------------|
| 데이터 | 136개 엑셀, 상류·다측정망 병합 | 자동측정망 10채널 wide |
| 프레임워크 | TensorFlow 2.4 | PyTorch |
| 채점 | imputed 라벨 포함 | obs-only 병행 |
| 한강 DO NSE | 0.915 (paper) | **-0.161** (obs) |

- 원논문 대비 재구현 median gap **~0.29 NSE**.
- Chronos 우위가 **5/20 셀**에서 원논문 기준 역전 가능.

---

### [10] Fair Comparison — **부분 공정**

**맞춘 것**: 동일 split, obs-only, 5일차 일평균, rolling origin.

**불리한 비대칭**:
- Chronos: pretrained + LoRA + ctx512 + 67지점 풀링 + HP 튜닝.
- GRU: 30 epoch, lr 고정, seed 42, 지점×타깃 독립, ctx 240.

---

### [11] Legacy Recovery — **미수행 (권고)**

- LR sweep, longer context, direct multi-horizon, seed variance 등 **미실시**.
- GRU << persistence는 **under-tuning 가능성**과 **구조적 한계**가 공존.

---

### [12] Historical Objective — **autoregressive-friendly 변형**

- 5일차 **일평균** 평가 → persistence에 유리.
- 120h 다스텝 vs persistence 평탄 유지 → DL 불리.
- 레거시 상류 병합·다측정망 목표가 **축소·단순화**됨.

---

### [13] Causal vs Pattern — **최종 분류**

| 검증 질문 | 결과 |
|-----------|------|
| 강우/유량이 forecast distribution을 바꾸는가? | **아니오** (운영 미사용) |
| uncertainty가 event 전에 증가하는가? | **부분적** (width ratio만, delta 적응 없음) |
| 극단 regime에서 calibration 유지? | **아니오** |
| sudden contamination adaptive response? | **아니오** |
| persistence 이상 행동? | **Chl-a 경보·일부 지점에서만** |

## 4. 치명적 실패 위험 (Critical Failure Risks)

1. **Conformal 후 quantile crossing** — 운영 확률구간으로 사용 불가.
2. **극단 subset NSE 붕괴** — 평균 성능이 위험을 가림.
3. **운영 경로 cov=None** — 환경 forcing 무시가 코드로 확정.
4. **`light_fill` 미래 오염** — 입력 leakage로 성능·신뢰도 과대 가능.
5. **GRU baseline strawman** — Chronos 우위 해석 과장 위험.

---

## 5. 숨겨진 실패 모드 (Hidden Failure Modes)

- `final_eval`(240h origin) vs `replay`(512h origin) **수치 불일치** — 보고서 간 직접 비교 위험.
- conformal 구현 **3종 혼재** (split CQR / online CQR / legacy 잔차분위).
- `p_exceed`는 외곽 4분위만 보정, 내부 quantile은 RAW 혼합.
- persistence 정의가 모듈마다 다름 (24h mean vs ffill 단일값).
- metric 함수가 `metrics.py` / `final_eval` / `prob_compare`에 **중복 정의**.

## 7. 권고 조치 (우선순위)

### P0 — 배포 차단/수정 필수

1. **Conformal 보정폭 음수 방지** + 보정 후 quantile monotonicity 재정렬.
2. **`light_fill` → origin-as-of causal fill** (`[0, o)` only).
3. **평가 origin 통일**: `test_origins(..., CTX)` = predict `context_length` 일치.

### P1 — 공정성·신뢰성

4. Alert 임계 percentile을 **val(+train) only**로 제한 (test obs 제외).
5. GRU baseline **HP retuning** (LR/epoch/seed/context/direct horizon).
6. 운영 경로에 **공변량 실연결** 후 counterfactual 재실험.

### P2 — 평가 체계

7. SLO를 **평균 NSE**가 아닌 **extreme-only / worst-10% / conditional coverage**로 전환.
8. metric·conformal 구현 **단일 모듈로 통합**.
9. AIHub **Test 폴더 holdout** 분리 또는 문서 명확화.
